Introduction

Bienvenue dans la documentation de l'API SMS Gateway. Cette API vous permet d'envoyer des SMS, de gérer plusieurs cartes SIM, et d'automatiser le basculement entre opérateurs.

URL de base

Production: https://votre-domaine.com/api
Développement: http://168.231.85.4:3001/api

Fonctionnalités principales

  • ✅ Envoi de SMS via multiple cartes SIM
  • ✅ Gestion automatique des quotas
  • ✅ Basculement automatique entre opérateurs
  • ✅ Historique complet des messages
  • ✅ Monitoring en temps réel
  • ✅ API RESTful complète

Formats supportés

L'API accepte et retourne du JSON. Assurez-vous d'inclure le header Content-Type: application/json dans vos requêtes.

Authentification

Actuellement, l'API ne nécessite pas d'authentification pour les environnements de développement. En production, nous recommandons d'implémenter une authentification par clé API.

Exemple d'authentification future (recommandé)

curl -X POST "https://api.smsgateway.com/api/messages/send" \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{"recipient": "+33612345678", "message": "Hello World"}'

API Messages

POST /api/messages/send

Envoie un SMS via la SIM par défaut ou une SIM spécifique.

Paramètres

Paramètre Type Requis Description
recipient string Oui Numéro de téléphone au format international (+33612345678)
message string Oui Contenu du message (max 160 caractères par SMS)
sim_id integer Non ID de la SIM à utiliser (utilise la SIM par défaut si omis)
priority string Non Priorité du message: "low", "normal", "high" (défaut: "normal")

Exemple de requête

{ "recipient": "+33612345678", "message": "Bonjour ! Votre commande #12345 a été expédiée.", "priority": "high" }

Réponse

{ "success": true, "message": "SMS envoyé avec succès", "data": { "id": 1234, "recipient": "+33612345678", "message": "Bonjour ! Votre commande #12345 a été expédiée.", "sim_used": { "id": 32, "carrier_name": "LycaMobile", "phone_number": "+33745520050" }, "status": "sent", "created_at": "2025-05-26T22:30:00.000Z" } }
GET /api/messages/history

Récupère l'historique des messages envoyés.

Paramètres de requête

Paramètre Type Description
page integer Numéro de page (défaut: 1)
limit integer Nombre d'éléments par page (défaut: 50, max: 100)
status string Filtrer par statut: "pending", "sent", "failed"
sim_id integer Filtrer par SIM utilisée
date_from string Date de début (format: YYYY-MM-DD)
date_to string Date de fin (format: YYYY-MM-DD)

Exemple de requête

GET /api/messages/history?page=1&limit=20&status=sent&date_from=2025-05-01
GET /api/messages/pending

Récupère les messages en attente d'envoi pour un téléphone spécifique.

Paramètres de requête

Paramètre Type Requis Description
phone_id string Oui ID unique du téléphone
limit integer Non Nombre maximum de messages à récupérer (défaut: 10)
PUT /api/messages/status

Met à jour le statut d'un message.

Paramètres

Paramètre Type Requis Description
message_id integer Oui ID du message à mettre à jour
status string Oui Nouveau statut: "sent", "failed", "delivered"
error_message string Non Message d'erreur si status = "failed"

API SIMs

GET /api/sims

Récupère la liste de toutes les cartes SIM avec leurs quotas.

Réponse

{ "success": true, "data": [ { "id": 32, "phone_number": "+33745520050", "carrier_name": "LycaMobile", "is_active": true, "is_default": true, "monthly_limit": 30000, "messages_sent_month": 91, "recipients_monthly_limit": 50, "recipients_monthly": 10, "usage_percentage": 20.0 } ] }
PUT /api/sims/{sim_id}/config

Met à jour la configuration d'une SIM.

Paramètres

Paramètre Type Description
monthly_limit integer Limite mensuelle de SMS
recipients_monthly_limit integer Limite mensuelle de destinataires uniques
is_active boolean Activer/désactiver la SIM
is_default boolean Définir comme SIM par défaut
GET /api/sims/stats

Récupère les statistiques globales des SIMs.

API Auto-Switch

GET /api/autoswitch/status

Récupère le statut du système de basculement automatique.

Réponse

{ "success": true, "data": { "enabled": true, "status": "normal", "message": "Système fonctionnel", "default_sim": { "carrier_name": "LycaMobile", "phone_number": "+33745520050", "max_usage_percentage": 20.0 }, "available_sims": [], "config": { "quota_alert_threshold": 80, "quota_check_frequency": 180 } } }
PUT /api/autoswitch/config

Met à jour la configuration du système auto-switch.

Paramètres

Paramètre Type Description
auto_switch_enabled boolean Activer/désactiver le basculement automatique
quota_alert_threshold integer Seuil d'alerte en pourcentage (1-100)
quota_check_frequency integer Fréquence de vérification en secondes (min: 30)
sms_notifications_enabled boolean Activer les notifications SMS d'alerte
notification_phone_number string Numéro pour recevoir les notifications
POST /api/autoswitch/switch

Force un basculement manuel vers la meilleure SIM disponible.

Webhooks

Les webhooks vous permettent de recevoir des notifications en temps réel sur les événements importants.

Événements disponibles

  • message.sent - Message envoyé avec succès
  • message.failed - Échec d'envoi de message
  • sim.quota_alert - Quota SIM atteint
  • autoswitch.switched - Basculement automatique effectué
  • phone.disconnected - Téléphone déconnecté

Configuration des webhooks

Exemple de payload webhook

{ "event": "message.sent", "timestamp": "2025-05-26T22:30:00.000Z", "data": { "message_id": 1234, "recipient": "+33612345678", "sim_used": { "id": 32, "carrier_name": "LycaMobile" } } }

PHP

<?php class SMSGateway { private $baseUrl; public function __construct($baseUrl = 'http://168.231.85.4:3001/api') { $this->baseUrl = $baseUrl; } public function sendSMS($recipient, $message, $simId = null) { $data = [ 'recipient' => $recipient, 'message' => $message ]; if ($simId) { $data['sim_id'] = $simId; } $ch = curl_init(); curl_setopt($ch, CURLOPT_URL, $this->baseUrl . '/messages/send'); curl_setopt($ch, CURLOPT_POST, true); curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data)); curl_setopt($ch, CURLOPT_HTTPHEADER, [ 'Content-Type: application/json' ]); curl_setopt($ch, CURLOPT_RETURNTRANSFER, true); $response = curl_exec($ch); $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE); curl_close($ch); if ($httpCode === 200) { return json_decode($response, true); } else { throw new Exception('Erreur HTTP: ' . $httpCode); } } public function getHistory($page = 1, $limit = 50) { $url = $this->baseUrl . '/messages/history?page=' . $page . '&limit=' . $limit; $response = file_get_contents($url); return json_decode($response, true); } } // Utilisation $sms = new SMSGateway(); try { $result = $sms->sendSMS('+33612345678', 'Bonjour depuis PHP!'); echo 'SMS envoyé: ' . $result['data']['id']; } catch (Exception $e) { echo 'Erreur: ' . $e->getMessage(); } ?>

Python

import requests import json class SMSGateway: def __init__(self, base_url='http://168.231.85.4:3001/api'): self.base_url = base_url def send_sms(self, recipient, message, sim_id=None, priority='normal'): """Envoie un SMS via l'API Gateway""" data = { 'recipient': recipient, 'message': message, 'priority': priority } if sim_id: data['sim_id'] = sim_id try: response = requests.post( f'{self.base_url}/messages/send', json=data, headers={'Content-Type': 'application/json'} ) response.raise_for_status() return response.json() except requests.exceptions.RequestException as e: raise Exception(f'Erreur lors de l\'envoi: {e}') def get_history(self, page=1, limit=50, status=None): """Récupère l'historique des messages""" params = {'page': page, 'limit': limit} if status: params['status'] = status response = requests.get( f'{self.base_url}/messages/history', params=params ) response.raise_for_status() return response.json() def get_sims_status(self): """Récupère le statut des SIMs""" response = requests.get(f'{self.base_url}/sims') response.raise_for_status() return response.json() # Utilisation if __name__ == '__main__': gateway = SMSGateway() try: # Envoyer un SMS result = gateway.send_sms( recipient='+33612345678', message='Bonjour depuis Python!', priority='high' ) print(f"SMS envoyé avec l'ID: {result['data']['id']}") # Récupérer l'historique history = gateway.get_history(limit=10) print(f"Nombre de messages: {len(history['data'])}") except Exception as e: print(f"Erreur: {e}")

Gestion d'erreurs

Codes de statut HTTP

Code Signification Description
200 OK Requête réussie
400 Bad Request Paramètres invalides ou manquants
404 Not Found Ressource non trouvée
429 Too Many Requests Limite de taux dépassée
500 Internal Server Error Erreur interne du serveur

Format des erreurs

{ "success": false, "message": "Numéro de téléphone invalide", "error_code": "INVALID_PHONE_NUMBER", "details": { "field": "recipient", "provided": "123456", "expected": "Format international (+33612345678)" } }

Gestion des erreurs recommandée

// JavaScript const handleSMSError = (error) => { switch (error.error_code) { case 'INVALID_PHONE_NUMBER': return 'Le numéro de téléphone n\'est pas valide'; case 'SIM_NOT_AVAILABLE': return 'Aucune SIM disponible pour l\'envoi'; case 'QUOTA_EXCEEDED': return 'Quota mensuel dépassé'; case 'MESSAGE_TOO_LONG': return 'Le message est trop long'; default: return 'Erreur inconnue lors de l\'envoi'; } }; try { await sendSMS('+33612345678', 'Test'); } catch (error) { const userMessage = handleSMSError(error); console.error(userMessage); }

Retry et Circuit Breaker

Stratégie de retry recommandée

const sendSMSWithRetry = async (recipient, message, maxRetries = 3) => { for (let attempt = 1; attempt <= maxRetries; attempt++) { try { return await sendSMS(recipient, message); } catch (error) { if (attempt === maxRetries) throw error; // Backoff exponentiel const delay = Math.pow(2, attempt) * 1000; await new Promise(resolve => setTimeout(resolve, delay)); } } };
